## 问题与决策

- 环境中未安装 `pytest` 与 `rg`（ripgrep），因此测试验证采用仓库内置的 `python -m unittest discover -s tests`；文档扫描采用内置 grep/ast-grep 工具完成。
- “加载阶段可流式跑通”不等于“真全量训练可跑通”：文档中刻意把两者拆开描述，避免误导复现。

## 遗留风险

- “真全量训练/评估（`--max-nodes 0`）”未在本机验证：受 CSR 邻接矩阵与多份 dense 矩阵常驻内存影响，仍需在大内存机器上阶梯放量。
- Windows 下 Bash 风格的多行命令（反斜杠续行）对 CMD/PowerShell 并非可直接复制执行，文档中虽已部分用占位降低风险，但仍建议读者留意运行环境。
