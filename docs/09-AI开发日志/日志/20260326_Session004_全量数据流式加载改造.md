# 20260326 Session004 全量数据流式加载改造

## 1. 目标

- 让 `edane_full_pipeline.py --mode file` 在全量/大规模 CSV 上具备“可跑通”的工程能力：
  - 不因 **整读 CSV** 爆内存
  - 评估阶段不因 **枚举全边/建巨大集合** 爆内存
  - 支持将快照分桶临时文件写到指定磁盘（NVMe）

## 2. 变更前问题（快照）

- `src/edane_full_pipeline.py::read_csv_dict()` 会把 CSV 整体读入 `List[Dict]`
- `build_graph_from_files()` 会累积 `temporal_edges` 列表并排序，再构造每个快照的 `set`
- `sample_link_pairs()` 在稀疏图路径会 `triu().tocoo()` 并构造全量 `edge_set`

以上在全量 OAG（或大子图）场景下会触发内存峰值与长时间无反馈。

## 3. 变更内容

### 3.1 file 模式加载改为流式 + 分桶写盘

- 新增 `iter_csv_dict()`：流式 DictReader（带大文件进度输出）
- 新增 `build_graph_from_files_streaming()`：
  - pass1 扫描 edges：度数统计（支持 bounded 近似 Top-K）+ time 分位点 reservoir sampling
  - pass1b 扫描 features/labels：只加载选中节点
  - pass2 将 edges 按快照分桶写入二进制临时文件（`--partition-dir`）
  -（可选）pass2b 将 attr_updates 分桶写入 CSV
  - 返回 `batch_iter`（iterator），上层边迭代边训练/更新，避免一次性持有全部 batches

### 3.2 评估采样改为“只采样、不枚举全边”

- `sample_link_pairs()`：
  - 稀疏图路径改为基于 CSR 的随机邻居采样 + membership 检测
  - 避免 `coo`/`edge_set` 的全量边复制

### 3.3 CLI 增强（面向全量跑通）

在 `build_parser()` 增加：

- `--time-quantile-sample`
- `--partition-dir` / `--keep-partitions`
- `--node-selection-mode` / `--node-selection-capacity`
- `--stable-batches`

补充：

- `--no-all-results`：不更新项目根目录 `all_results.csv`，便于全量长跑不污染仓库

### 3.4 dry-run 统计改进

- `prepare_datasets.py --dry-run` 现在允许目标 CSV 已存在时运行（不会写出文件，只做统计），用于拿到 `kept_papers/written_edges`。

### 3.4 仓库卫生

- 增补 `.gitignore`：忽略 `__pycache__/`、`.cache/`、`*.pyc` 等
- 删除仓库中误提交的 `src/__pycache__/*.pyc`（避免运行时反复污染工作区）

## 6. 风险与后续

- **全量训练**仍受 CSR 邻接与 embedding 常驻内存限制；即便加载阶段不爆内存，也可能在训练/更新阶段 OOM。
- 若目标是“真正全量训练/评估（--max-nodes 0）”，需要在服务器上结合内存估算与渐进放量测试。
- 当前未实现断点续跑（checkpoint/resume）；如果后续跑全量需要更强鲁棒性，可考虑：
  - 按快照落盘模型状态（EDANE：adj/emb/attrs）
  - 基于分桶文件从指定 snapshot 继续
