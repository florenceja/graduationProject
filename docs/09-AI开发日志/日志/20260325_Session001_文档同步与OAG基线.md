## 6. 风险与遗留

1) 当前 OAG 基准是 proxy task：

- 引用边方向在流水线中会被规范化为无向图
- `venue` 作为单标签分类存在噪声与同名碎片化风险

2) 全量 OAG：

- （当时结论）转换可在更强硬件上完成；且当时 file 模式尚未实现“流式扫描 + 分桶写盘”，加载阶段会先整体载入 CSV
   - 现状更新：已在后续 Session004 将 file 模式加载改为“流式扫描 + 快照分桶写盘”；但训练/更新阶段仍受 CSR 邻接矩阵与 embedding 常驻内存限制，通常仍需要 `--max-nodes` 渐进放量

3) 本地目录清理：

- `data/amazon2m_sample/` 可能因 Windows 文件占用无法删除，需要关闭占用进程后再清理

## 7. 下一步建议

- 若要把“全量 OAG”作为论文复现能力，建议：全量转换只做索引/统计 → 再采样出可训练子图
- 若要在本机获得更可信的图结构实验，优先优化子图构造策略（例如 seed + k-hop 引用扩展）
